## Overview

This application provides a complete user management system with role-based access control. It supports three user roles (Super Admin, Admin, and Normal User) with different permissions and access levels.

### Role Permissions

- **Super Admin**: Can manage all users, including other admins
- **Admin**: Can manage normal users, view other admins
- **Regular User**: Can only view and edit their profile

## Features

- **Authentication System**
  - Login with role-based redirection
  - Signup page (creates normal users)
  - JWT-based authentication (simulated)

- **Role-Based Dashboards**
  - Super Admin Dashboard with complete management capabilities
  - Admin Dashboard with limited management capabilities
  - User Dashboard with personal profile management

- **User Management**
  - User list with pagination
  - Filter users by role and status
  - Search users by username or email
  - Create/Edit/Delete users based on permissions
  - Activate/Deactivate users

- **Profile Management**
  - View and edit personal information
  - Change password functionality

- **Responsive Design**
  - Mobile-friendly interface
  - Adaptive sidebar
  - Dark Mode/Light Mode

## API Documentation

This implementation uses mock services to simulate API calls. In a real-world scenario, these would be replaced with actual API calls to a backend server.

### Authentication Endpoints

#### Login

- **Method**: POST
- **URL**: `/api/auth/login`
- **Body**: `{ email: string, password: string }`
- **Response**: `{ user: User, token: string }`


#### Signup

- **Method**: POST
- **URL**: `/api/auth/signup`
- **Body**: `{ username: string, email: string, firstName: string, lastName: string, password: string }`
- **Response**: `{ success: boolean }`


#### Get Current User

- **Method**: GET
- **URL**: `/api/auth/me`
- **Headers**: `Authorization: Bearer <token>`
- **Response**: `User`


### User Management Endpoints

#### Get Users

- **Method**: GET
- **URL**: `/api/users?page=1&limit=10&search=&role=&status=`
- **Headers**: `Authorization: Bearer <token>`
- **Response**: `{ users: User[], totalUsers: number, totalPages: number, currentPage: number }`


#### Get User by ID

- **Method**: GET
- **URL**: `/api/users/:id`
- **Headers**: `Authorization: Bearer <token>`
- **Response**: `User`


#### Create User

- **Method**: POST
- **URL**: `/api/users`
- **Headers**: `Authorization: Bearer <token>`
- **Body**: `User`
- **Response**: `User`


#### Update User

- **Method**: PUT
- **URL**: `/api/users/:id`
- **Headers**: `Authorization: Bearer <token>`
- **Body**: `User`
- **Response**: `User`


#### Delete User

- **Method**: DELETE
- **URL**: `/api/users/:id`
- **Headers**: `Authorization: Bearer <token>`
- **Response**: `{ success: boolean }`


#### Update User Status

- **Method**: PATCH
- **URL**: `/api/users/:id/status`
- **Headers**: `Authorization: Bearer <token>`
- **Body**: `{ status: "ACTIVE" | "INACTIVE" }`
- **Response**: `User`
